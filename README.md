# 🎖️ AI Storyclass - 소나기 (디지털 일기 포토북)

이 프로젝트는 사용자의 소중한 일기나 기록을 AI 기술을 활용하여 한 권의 디지털 포토북으로 만들어주는 서비스입니다. Sweetbook API를 활용하여 실제 인쇄 가능한 수준의 도서를 설계하고 관리할 수 있습니다.

---

## 1. 과제 수행 과정

### 🎯 접근 방식
처음 과제를 받았을 때, 단순한 게시판 형태보다는 **'실제 책을 넘겨보는 듯한 경험'**을 주는 것이 가장 중요하다고 생각했습니다. 따라서 3D Flip 효과를 핵심 UI로 설정하고 개발을 시작했습니다.

### 🛠 작업 순서
1.  **UI 기획 및 설계**: Tailwind CSS와 Vue 3를 사용하여 현대적이고 깔끔한 화이트톤의 갤러리 스타일 UI를 설계했습니다.
2.  **핵심 컴포넌트 개발**: 책의 앞표지, 내지, 뒷표지가 자연스럽게 넘어가는 `BookViewer` 컴포넌트를 구현했습니다.
3.  **API 연동**: Sweetbook API를 연동하여 실제 도서 생성, 콘텐츠(페이지) 추가, 주문 접수 기능을 구현했습니다.
4.  **백엔드 구축**: Express.js를 사용하여 로컬 데이터베이스(JSON 기반)와 API를 중계하는 서버를 구축했습니다.
5.  **권한 및 제한 로직**: 관리자(Admin)는 모든 도서를 보고, 일반 사용자는 자신의 도서 1권만 관리할 수 있는 로직을 추가했습니다.

### ⏱ 시간 배분
*   UI/UX 및 3D Flip 로직 구현: 40%
*   백엔드 API 서버 구축 및 데이터베이스 설계: 30%
*   API 연동 및 권한 로직 고도화: 20%
*   테스트 및 버그 수정 (포트 충돌 해결 등): 10%

## 2. API를 써보고 느낀 점

### 👍 좋았던 점
*   샌드박스 API 키를 통해 실제 비용 결제 없이도 도서 생성부터 주문까지의 전 과정을 테스트해볼 수 있어 개발 접근성이 매우 좋았습니다.
*   API 문서가 직관적이며, 특히 `parameters` 구조를 통해 텍스트와 이미지를 유연하게 배치할 수 있는 점이 인상적이었습니다.

### 🛠 개선되었으면 하는 점
*   `templateUid`를 선택할 때, 해당 템플릿이 어떤 레이아웃을 가지고 있는지(이미지 영역, 텍스트 영역의 위치 등)를 미리 알 수 있는 **'템플릿 미리보기 API'**가 있다면 개발자가 더 적절한 디자인을 선택하기 수월할 것 같습니다.
*   도서 규격(`bookSpec`)과 사용 가능한 템플릿 사이의 의존 관계가 더 명확하게 가이드되면 좋겠습니다.

## 3. 과제에서 내린 가장 중요한 판단

### 🧐 상황: UI 방식의 결정
단순한 그리드 형태의 일기 목록을 보여줄 것인가, 아니면 실제 책 모양의 뷰어를 만들 것인가에 대한 선택이 필요했습니다.

### 💡 선택지 및 결정 이유
*   **선택지 1**: 대중적인 피드(Feed) 스타일 (구현 속도 빠름, 익숙함)
*   **선택지 2**: 3D 책 넘김 방식 (구현 복잡도 높음, '도서' 컨셉에 최적화)
*   **결정**: **선택지 2**를 택했습니다. '스윗북'이라는 서비스의 본질이 '책을 만드는 것'에 있으므로, 사용자에게 자신의 기록이 실제 책으로 제작되었을 때의 느낌을 미리 시뮬레이션해주는 것이 가장 큰 가치라고 판단했습니다.

### ✅ 결과
사용자로부터 "진짜 내 일기가 한 권의 책이 된 것 같다"는 시각적 만족감을 줄 수 있게 되었으며, 서비스의 정체성을 훨씬 명확하게 전달할 수 있었습니다.

## 4. AI 도구를 사용하면서 겪은 문제 해결

### ❌ 문제 상황
윈도우 환경에서 백엔드 서버를 4000번 포트로 실행하려 했으나 `EADDRINUSE` 오류가 발생했습니다.

### 🤖 AI의 답변과 한계
AI는 `netstat -ano`를 통해 PID를 찾고 `taskkill`로 프로세스를 종료하라는 전형적인 답변을 주었습니다. 하지만 윈도우의 특정 시스템 프로세스나 좀비 프로세스가 포트를 잡고 있는 경우 PID가 목록에 나타나지 않아 해결되지 않았습니다.

### 💡 해결 과정
AI에게 "PID가 검색되지 않는데도 포트 충돌이 난다"고 구체적인 상황을 다시 질문했습니다. AI는 "윈도우 예약 포트 문제일 수 있다"며 포트 번호 자체를 변경하는 것을 제안했습니다.

### 🎯 최종 해결
백엔드 포트를 **4001**로 변경하고 프론트엔드의 호출 주소를 일괄 업데이트하여 문제를 즉시 해결했습니다. 이를 통해 환경적 제약에 갇히기보다 유연한 우회책을 찾는 것이 실무에서 더 효율적일 수 있음을 다시 깨달았습니다.

## 🚀 실행 방법

### 백엔드 (Backend)
```bash
cd backend
npm install
npm run dev
```
*포트 4001번에서 실행됩니다.*

### 프론트엔드 (Frontend)
```bash
cd frontend
npm install
npm run dev
```
*Vite 기본 포트(5173 등)에서 실행됩니다.*
